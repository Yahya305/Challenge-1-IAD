We Are Group 9

During Lab Session 1, our objective was to develop a system that generates 10 random math problems while tracking response times and enforcing a fixed time limit (e.g., 5 seconds) per question. Incorrect answers incurred time penalties, discouraging random guessing. If the user failed to respond within the given time, the problem was marked incorrect. At the end of the session, the system calculated and displayed the total correct answers, total time taken (including penalties), and average time per problem. This approach enhanced real-time problem-solving skills while reinforcing accuracy and time management.

Frequently Asked questions

1

What happens if I enter non-numeric characters?

The input field is designed to accept only numeric values. If you attempt to enter non-numeric characters, the border of the input field will turn red, and an error placeholder message will appear. Your answer will not be accepted until you enter a valid number.

2

What's the difference between timeout and interval?

In this quiz, a timeout refers to the overall time limit per question, which is set at 5 seconds. If the time runs out, the answer is automatically submitted. An interval, on the other hand, updates the visual countdown timer every second, ensuring that users see a real-time countdown. While penalties reduce the remaining time, they do not affect the frequency of the visual countdown updates.

3

What happens if I answer incorrectly?

If you submit an incorrect answer, a 2-second time penalty is applied, and the input field is reset. If the penalty causes the remaining time to drop to zero or below, the question ends immediately. Additionally, any penalty seconds are added to your final total time.
